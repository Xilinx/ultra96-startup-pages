Ultra96-V2 Usages

Below is a list of ways that you can access the Ultra96-V2 Board. These are explained in depth in their own sections below.


  • Webapp

  • miniDP

  • UART

  • SSH

Webapp

This is the simplest of all of the methods. When the board boots for the first time or any other time (into Linux) the webapp will be available by default. Using any device that is able to use Wi-Fi (laptop, tablet, cellphone, etc.) can connect to the webapp by connecting to the board. When the board boots it shows its MAC address that is appended to the string Ultra96-V2_ which is what the SSID (network) that can be connected to. . If you happen to forget it is 192.168.2.1 and that will bring you to the home page of the webapp.

The webapp has many uses from running examples to creating your own webpages/projects. Feel free to play around and explore all that the webapp has to offer.

miniDP

It is also possible to see the Linux desktop when you boot the board. This is done by connecting a miniDP to DP cable to the Ultra96-V2 kit. Alternatively you can use a miniDP to X adapter where X is some other standard to connect to the monitor, such as HDMI. Once this is done you just simply boot up the board and after a short boot sequence you will see the desktop. From here you can open up the terminal and/or run any other applications as you would in any other Linux environment.

The following "active" miniDP to HDMI adapters are known to work with the Ultra96-V2 board:

  • Ableconn MDPHD4KAA
  • CableCreation CD0095
  • Cable Matters 101021

Note that "passive" style adapters will not work.

UART

If you are fine with using just the terminal and have a microUSB to USB cable and an Avnet USB UART/JTAG adapter then using this option will work. Connect the microUSB into the port on the USB/JTAG adapter (sold separately) and connect the USB into a laptop (or other computer). It is recommended to use either PuTTY or TeraTerm in order to communicate to the board. Both TeraTerm and PuTTY can be downloaded online for free. The explanation provided will assume PuTTY has been chosen (TeraTerm works close enough that it shouldn't be too much hassle to use that instead) and that it is being used on a Windows OS.

Once downloaded and everything is connected (and the board is on) open up Device Manager in Windows by typing it into the search bar after pressing the Windows Icon. Scroll down to Ports (COM & LPT) and find the USB Serial Port (COM<X>) where X is a number. If you have many of them just unplug the USB cable from your device and see what went away and plug it back in. Remember the number X. To find the COM Port in other OS is simple enough and can be found out by searching for the way how online.

Open up PuTTY and select Serial as the Connection Type:. In the space under Serial line enter the COM port that you saw before (e.g. COM5). Make sure the Speed is set to 115200. Once all of this is ready just press the Open button at the bottom of the application and you will get a terminal open. Congratulations, you now have access to the Ultra96-V2 terminal.

SSH

In order to SSH into the board it is once again recommended to use PuTTY or TeraTerm. We will assume for this tutorial that PuTTY is used on a Windows OS. Both programs can be downloaded for free online.

Connect to the Ultra96-V2 board via its SSID. The board's MAC address is appended to the string "Ultra96-V2_" (e.g. Ultra96-V2_AABBCCDDEEFF) which is the SSID (network) that can be connected to. Connect to this network. Next open up PuTTY and under Connection type: select SSH. Under Host Name (or IP address) enter the static IP address for the Ultra96-V2 board (192.168.2.1) with the Port set to 22. Click the Open button on the bottom of the page and the terminal of the Ultra96-V2 should open appropriately.
